## Known Limitations

1. **Authentication is minimal** — JWT auth exists without user registration UI
2. **No built-in model hosting** — requires external LLM API keys
3. **MCP integrations require individual tokens** — leave empty to skip
4. **No multi-tenancy** — single organization per deployment
5. **Tracing exports to console only** — no OTLP collector by default
6. **No horizontal scaling** — single-instance Docker Compose deployment
7. **Default storage credentials** — `minioadmin/minioadmin` for MinIO
8. **Database schema evolving** — placeholder tables in init.sql
9. **No CI/CD pipeline** — manual deployment only
10. **License marked Private** — not yet open-source ready

## Recommended Next Steps

1. **Set `JWT_SECRET_KEY`** — generate with `python -c "import secrets; print(secrets.token_hex(32))"` before any production use
2. **Change default passwords** — update POSTGRES_PASSWORD, REDIS_PASSWORD, MINIO credentials
3. **Configure CORS** — set `CORS_ALLOWED_ORIGINS` for your deployment domain
4. **Add LLM provider keys** — at minimum `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`
5. **Run demo seed** — `docker compose exec backend python -m database.docker.seed_demo`
6. **Import Grafana dashboards** — point Grafana at Prometheus (see `monitoring/grafana/README.md`)
7. **Enable alerting** — wire `alerting_rules.yaml` into Prometheus (see `monitoring/alerts/README.md`)
8. **Consider open-sourcing** — update LICENSE from "Private" if applicable
